Class Definitions

"dtAlignmentObject" Class

The AlignmentObject is an alias for schema.org’s AlignmentObject and uses IRIs from that vocabulary.

The list and definition of the child properties is:-

targetName
Name of the alignment.
targetUrl
URL linking to the official description of the alignment target, for example an individual standard within an educational framework.
targetDescription
Short description of the alignment target.
targetFramework
Name of the framework the alignment target.
targetCode
If applicable, a locally unique string identifier that identifies the alignment target within its framework and/or targetUrl.

"dtAssertion" Class

Assertions are representations of an awarded badge, used to share information about a badge belonging to one earner.

The list and definition of the child properties is:-

id
Unique IRI for the Assertion. If using hosted verification, this should be the URI where the assertion is accessible. For signed Assertions, it is recommended to use a UUID in the urn:uuid namespace.
type
Valid JSON-LD representation of the Assertion type. In most cases, this will simply be the string Assertion. An array including Assertion and other string elements that are either URLs or compact IRIs within the current context are allowed.
recipient
The recipient of the achievement.
badge
IRI or document that describes the type of badge being awarded. If an HTTP/HTTPS IRI The endpoint should be a BadgeClass.
verification
Instructions for third parties to verify this assertion. (Alias 'verify' may be used in context.)
issuedOn
Timestamp of when the achievement was awarded.
image
IRI or document representing an image representing this user’s achievement. This must be a PNG or SVG image, and should be prepared via the Baking specification. An ‘unbaked’ image for the badge is defined in the BadgeClass and should not be duplicated here.
evidence
IRI or document describing the work that the recipient did to earn the achievement. This can be a page that links out to other pages if linking directly to the work is infeasible.
narratrive
A narrative that connects multiple pieces of evidence. Likely only present at this location if evidence is a multi-value array.
expires
If the achievement has some notion of expiry, this indicates a timestamp when a badge should no longer be considered valid. After this time, the badge should be considered expired.
revoked
Defaults to false if Assertion is not referenced from a revokedAssertions list and may be omitted. See RevocationList. If revoked is true, only revoked and id are required properties, and many issuers strip a hosted Assertion down to only those properties when revoked.
revocationReason
Optional published reason for revocation, if revoked.
endorsement
extensions

"dtAssertionsResponse" Class

Responses from all APIs in this document are wrapped in a JSON envelope.

The list and definition of the child properties is:-

status
The 'status' property MUST appear on all responses.
results
A list of zero or more assertions responsive to the request.

"dtBadgeClass" Class

A collection of information about the accomplishment recognized by the Open Badge.

The list and definition of the child properties is:-

id
Unique IRI for the BadgeClass. Most platforms to date can only handle HTTP-based IRIs. Issuers using signed assertions are encouraged to publish BadgeClasses using HTTP IRIs but may instead use ephemeral BadgeClasses that use an id in another scheme such as urn:uuid.
type
Valid JSON-LD representation of the BadgeClass type. In most cases, this will simply be the string BadgeClass. An array including BadgeClass and other string elements that are either URLs or compact IRIs within the current context are allowed.
name
The name of the achievement.
description
A short description of the achievement.
image
IRI of an image representing the achievement. May be a Data URI, or URI where the image may be found.
criteria
URI or embedded criteria document describing how to earn the achievement.
issuer
IRI or document describing the individual, entity, or organization that issued the badge.
alignment
An object describing which objectives or educational standards this badge aligns to, if any.
tags
A tag that describes the type of achievement.
related
version
The version identifier for the present edition of the entity.
endorsement
extensions

"dtClaim" Class

The claim the endorser is making.

The list and definition of the child properties is:-

id
The id of the entity the endorser would like to claim about.
endorsementComment
The simple claim the endorser is making about the entity.
extensions
The properties of the entity the endorser is making a claim about.

"dtCriteria" Class

Descriptive metadata about the achievements necessary to be recognized with an Assertion of a particular BadgeClass.

The list and definition of the child properties is:-

id
The URI of a webpage that describes in a human-readable format the criteria for the BadgeClass.
type
Defaults to Criteria.
narrative
A narrative of what is needed to earn the badge.

"dtCryptographicKey" Class

Alias for the Key class from the W3C Web Payments Community Group Security Vocabulary. A CryptographicKey document identifies and describes a Key used for signing Open Badges documents.

The list and definition of the child properties is:-

type
CryptographicKey.
id
The identifier for the key. Most platforms only support HTTP(s) identifiers.
owner
The identifier for the Profile that owns this key. There should be a two-way connection between this Profile and the CryptographicKey through the owner and publicKey properties.
publicKeyPem
The PEM key encoding is a widely-used method to express public keys, compatible with almost every Secure Sockets Layer library implementation.

"dtIdentityObject" Class

A collection of information about the recipient of a badge.

The list and definition of the child properties is:-

identity
Either the hash of the identity or the plaintext value. If it’s possible that the plaintext transmission and storage of the identity value would leak personally identifiable information where there is an expectation of privacy, it is strongly recommended that an IdentityHash be used.
type
The property by which the recipient of a badge is identified. This value should be an IRI mapped in the present context. For example, email maps to http://schema.org/email and indicates that the identity of the IdentityObject will represent a value of a Profile’s email property. Currently the only supported value for many platforms that implement Open Badges is email, but other properties are available.
hashed
Whether or not the identity value is hashed.
salt
If the recipient is hashed, this should contain the string used to salt the hash. If this value is not provided, it should be assumed that the hash was not salted.

"dtImage" Class

Metadata about images that represent Assertions, BadgeClasses or Profiles.

The list and definition of the child properties is:-

type
Defaults to ImageObject.
id
The URI or Data URI of the image.
caption
The caption for the image.
author
The profile id of the author of the image, if desired.

"dtMarkdown Text" Class

Text that may contain formatting according to Markdown syntax. Due to uneven support in displayers, publishers are encouraged to limit usage to simple elements like links, emphasis, and lists. Displayers may choose a subset of Markdown formatting to support or how to render this field. Images and tables may or may not be supported.

"dtProfile" Class

A Profile is a collection of information that describes the entity or organization using Open Badges. Issuers must be represented as Profiles, and recipients, endorsers, or other entities may also be represented using this vocabulary. Each Profile that represents an Issuer may be referenced in many BadgeClasses that it has defined. Anyone can create and host an Issuer file to start issuing Open Badges. Issuers may also serve as recipients of Open Badges, often identified within an Assertion by specific properties, like their url or contact email address. An Issuer Profile is a subclass of the general Profile with some additional requirements.

The list and definition of the child properties is:-

id
Unique IRI for the Issuer/Profile file. Most platforms to date can only handle HTTP-based IRIs.
type
Valid JSON-LD representation of the Issuer or Profile type. In most cases, this will simply be the string Issuer or the more general Profile. An array including Issuer and other string elements that are either URLs or compact IRIs within the current context are allowed.
name
The name of the entity or organization.
url
The homepage or social media profile of the entity, whether individual or institutional. Should be a URL/URI Accessible via HTTP.
telephone
A phone number for the entity. For maximum compatibility, the value should be expressed as a + and country code followed by the number with no spaces or other punctuation, like +16175551212 (E.164 format).
description
A short description of the issuer entity or organization.
image
An image representing the issuer. May be a Data URI, or URI where the image may be found or an instance of the Image class.
email
Contact address for the individual or organization.
publicKey
The key(s) an issuer uses to sign Assertions.
verification
Instructions for how to verify Assertions published by this Profile.
revocationList
HTTP URI of the Badge Revocation List used for marking revocation of signed badges.
endorsement
extensions

"dtRevocationList" Class

The Revocation List is a document that describes badges an Issuer has revoked that used the signed verification method.

The list and definition of the child properties is:-

type
RevocationList.
id
The id of the RevocationList.
issuer
The id of the Issuer.
revokedAssertions
A string id or UID-based identification of a badge object that has been revoked.

"dtStatus" Class

Response status.

The list and definition of the child properties is:-

error
A nullable string and the human-readable message describing the problem.
statusCode
The HTTP status code of the response.
statusText
A string matching one of the enumerated and allowed values for the given endpoint.

"dtVerificationObject" Class

A collection of information allowing an inspector to verify an Assertion.

The list and definition of the child properties is:-

type
The type of verification method.
verificationProperty
The @id of the property to be used for verification that an Assertion is within the allowed scope.
startsWith
The URI fragment that the verification property must start with.
allowedOrigins
The host registered name subcomponent of an allowed origin.
creator
The (HTTP) id of the key used to sign the Assertion.
