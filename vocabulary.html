This document describes a number of classes and properties that can be used to express authenticable claims by issuers about recipients in the form of Linked Data credentials. Each credential makes claims about a recipient’s qualifications or qualities. These claims represent either achievement of a particular defined benchmark as assessed by the issuer, or one or more properties of the recipient. When recorded and transmitted according to this specification, credentials are possible to be validated as authentic expressions of issuer’s trust in credential recipients.

This is an experimental document that is being actively deveveloped by the W3C Credentials Community Group.

This entire document is a work in progress in very early draft stage. It is expected to evolve considerably.

Introduction

This document describes a number of classes and properties that can be used to express authenticable claims by issuers about recipients in the form of Linked Data credentials. Each credential makes claims about a recipient’s qualifications or qualities. These claims represent either achievement of a particular defined benchmark as assessed by the issuer, or one or more properties of the recipient. When recorded and transmitted according to this specification, credentials are possible to be validated as authentic expressions of issuer’s trust in credential recipients.

The terms defined here serve use cases identified by the Credentials Community Group.

Glossary

Credential

A claim from an entity(s) that denotes qualification, achievement, quality, or other piece of information about an entity’s background such as a name, government ID, payment provider, home address, or university degree.

Issuer

An entity that creates and assigns credentials to recipients. It may be expressed as an Identity document.

This and potentially a couple other terms may need to be moved down or duplicated as a class, since we're referencing it as a class that may be included in credentials.

Recipient

Also known as the credentialee, an entity that is in control of a particular credential; who is typically both the primary subject of the information in a credential and the entity that initiates its transmission to a consumer. It may be expressed as an Identity document.

Consumer

An entity that is the end user of a credential. Consumers read, validate, and take action based on credentials.

Requestor

An entity that requests a credential from a recipient.

Audience

A consumer entity who is presented a credential, presumably by its recipient.

User Agent

A program, such as a browser or other Web client, that mediates the communication between parties on behalf of an issuer, recipient or consumer, such as the process of negotiating the response to a requestor’s request for a particular type of credential on behalf or a recipient.

Credential Management Service

A program, such as a credential storage vault, backpack, or personal credential wallet, that stores and protects access to an credentialee's credentials and can, on request, provide secure access to Requestors. The software process involved is known as the Credential Agent.

Credential Agent

A program that mediates secure access of a credentialee’s credentials for a requestor on behalf of the credential management service.

Classes

Identity

An Identity is a document representing an entity (that could serve as Issuer, Recipient, or Consumer of credentials). It is defined in terms of descriptive properties and recognized achievements. Together, these describe the entity and list the verifiable credentials that assert those properties and achievements.

Status
unstable
Parent Class
Class
Common Properties
name, description, url, image, birthdate, email, credential, Defined Achievement

The following example shows an Identity document, making its own claims about properties of the subject. 

{
  "@context": "https://w3id.org/credentials/v1",
  "id": "https://example.com/identities/bob",
  "type": ["Identity", "Person"],
  "name": "Bob Bobman",
  "birthdate": "1985-12-14",
  "email": "bbob@example.com"
}

Credential

A verifiable set of claims made by an issuer about a recipient, expressed in terms of properties claimed about the recipient and defined achievements conferred on the recipient.

Status
unstable
Parent Class
Class
Common Properties
type, claim, expirationDate, signature

The following example is a signed Credential issued by Big University and received by Bob, claiming a name and a birthdate. The type of credential must be a fully qualified IRI or dereferencable to an IRI in the document context. 

{
  "@context": "https://w3id.org/credentials/v1",
  "credential": [{
    "id": "http://biguniversity.gov/credentials/53091",
    "type": "BasicInfo",
    "claim": {
      "id": "https://example.com/identities/bob",
      "name": "Bob Bobman",
      "birthdate": "1985-12-14"
    },
    "expires": "2018-01-01",
    "signature": {
       "type": "GraphSignature2012",
       "creator": "https://biguniversity.edu/keys/1",
       "signature": "3780eyfh3q0fhhfiq3q9f8ahsidfhf29rhaish"
    }
  }]
}

Claim Set

The claims made by a Credential are packaged as a Claim Set, a set of properties asserted about the recipient. Each claim is a property-value pair, where the issuer asserts one quality or attribute of a credential recipient or of an achievement attributable to the earner. Each property name takes the form of an IRI, dereferencable within the document context to a term definition.

Status
unstable
Parent Class
Credential
Common Properties
id, badge, claims

The following example, which would appear as a component of a Credential is a set of claims defining the birthdate and governmentId number that apply to a particular recipient identity. The IRI of the recipient identity is the id (aliased in the context to JSON-LD keyword "@id") of the claim set property. Each of the claim terms that appear here are mapped to an IRI in the context. Otherwise, they would appear as full IRIs.

What property types are really going to appear in the credentials context? Do birthdate and governmentId make the cut? 

{
  "@context": "https://w3id.org/credentials/v1"
  ...
  "claim": {
    "id": "https://example.org/identities/alice",
    "birthdate": "1968-10-14",
    "governmentId": "123-99-2830"
  }
}

Defined Achievement

A Defined Achievement (also known as a Badge Class) is a specific claim intended to be made about many recipients. A credential class may be specific to one issuer or shared between multiple issuers. It is defined with a name, description, may link to a page describing detailed criteria, and may be visually represented by an image. An issuer property may point to the entity that defined the achievement.

Status
unstable
Parent Class
Defined Achievement Assertion
Common Properties
name, description, image, criteria, Issuer, alignment, tags

Would Defined Achievement Assertion really be the parent class here? An instance of DAA would appear as a document parent of an instance of this class, but I don't know if that's really what makes the relationship.

The following example shows a defined achievement that an example model rocketry club established to issue to recipients who met a predetermined level of expertise in rocketry, as assessed within that organization's context. 

{
  "@context": "https://w3id.org/credentials/v1",
  "id": "https://modelrockets.example.com/badges/l3r",
  "type": "BadgeClass",
  "name": "Level 3 Model Rocketeer",
  "description": "Recipient has demonstrated a thorough understanding of rocket safety procetures and rocket construction techniques",
  "image": "https://modelrockets.example.com/badges/l3r/image",
  "criteria": "https://modelrockets.example.com/badges/l3r/criteria",
  "issuer": {
    "id": "https://modelrockets.example.com/id",
    "name": "Model Rockets Example Club",
    "url": "https://modelrockets.example.com"
  }
}

Defined Achievement Assertion

An instance of a Defined Achievement as it applies to one issuer. A Defined Achievement may be added to a Credential within a Defined Achievement Assertion.

Status
unstable
Parent Class
badgeAssertion
Common Properties
image, Issuer, Defined Achievement

This example shows a "Bachelor of Arts" Defined Achievement as issued in a Credential and displayed within an Identity document. It appears as a child of a badgeAssertion claim type. the badge IRI ("id") value also appears in the Identity's "badge" property as an easily-queried listing of defined achievmements received by this identity. 

{
  "@context": "https://w3id.org/credentials/v1",
  "id": "https://example.com/identities/bob",
  "type": ["Identity", "Person"],
  "name": "Bob Bobman",
  "birthdate": "1985-12-14",
  "email": "bbob@example.com"
  "badge": ["https://bigunivertsity.edu/credentials/BA"],
  "credential": [{
    "id": "http://biguniversity.gov/credentials/53091",
    "type": "UniversityDegree",
    "claim": {
      "id": "https://example.com/identities/bob",
      "name": "Bob Bobman",
      "birthdate": "1985-12-14",
      "badgeAssertion": [
        {
          "id": "https://biguniversity.edu/credentials/53091/badge",
          "type": "Assertion",
          "recipient": {
            "identity": "bbob@example.com",
            "type": "email",
            "hashed": false
          },
          "issueDate": "2015-03-01",
          "badge": {
            "id": "https://bigunivertsity.edu/credentials/BA",
            "type": "BadgeClass",
            "name": "Bachelor of Arts",
            "description": "The gold standard of credentials in liberal arts education",
            "image": "https://biguniversity.edu/credentials/BA/image",
            "criteria": "https://biguniversity.edu/credentials/BA/criteria",
            "issuer": {

Properties

claim

A claim is the statement of trust an issuer makes about a credential recipient. A set of property value pairs, where in each property the issuer asserts one quality or attribute of a credential earner or of an achievement attributable to the earner. Each claim property name is dereferencable to a term definition.

The following example expresses an set of claims about an individual. The claim set has the id (here aliased to the JSON-LD keyword "@id") of the identity that is the recipient of the credential and the subject of the claims. Each shorthand claim ("image" and "name") have keywords that are terms defined in the document context that map these shorthand names to full IRIs. The claim about the subject's color uses a fully qualified IRI. 

{
  "@context": ["https://w3id.org/credentials/v1", "https://w3id.org/identity/v1"],
  "id": "http://example.com/issuedCredentials/5",
  "type": "Credential",
  "claim": {
    "id": "http://example.org/users/kittensworth"
    "image": "http://placekitten.com/100/100/",
    "name": "Sir Kittensworth",
    "https://schema.org/color": "chestnut"
  }
}
