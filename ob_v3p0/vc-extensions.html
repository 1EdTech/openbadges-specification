Verifiable Credentials Extensions

The [[[VC-DATA-MODEL]]] standard defines several types of extensions to enable "permissionless innovation". Some of the extensions are tracked in the [[[VC-EXTENSION-REGISTRY]]]. VC extensions fall into 6 categories:

  • Proof Methods
  • Status Methods
  • Data Schema Validation Methods
  • Refresh Methods
  • Terms of Use Methods
  • Evidence Methods

This standard defines three VC extensions: a Status Method, a Refresh Method, and a Data Schema Validation Method. Future versions of this specification may define additional extensions.

RevocationList

The "RevocationList" Status Method is used by a [=verifier=] to determine if the [=verifiable credential=] has been revoked. Or if a previously revoked credential has been reinstated. This extension provides one of the benefits of Hosted Verification in [[OB-20]].

Determine if a Credential is Revoked

If the credentialStatus property is present, and the type of the CredentialStatus object is "RevocationList", the [=verifier=] SHOULD follow these steps:

  1. Fetch the RevocationList from the id of the CredentialStatus object.
  2. If the id of the credential being verified appears in the list of revokedCredentials and the corresponding revoked property is true or ommitted, then the credential has been revoked by the issuer.

CredentialRefresh

The "CredentialRefresh" Refresh Method is used by a [=verifier=] to retrieve the current version of a [=verifiable credential=]. This extension provides one of the benefits of Hosted Verification in [[OB-20]].

Refresh a Credential

If the credentialRefresh property is present, and the type of the CredentialRefresh object is "CredentialRefresh", the [=verifier=] SHOULD follow these steps:

  1. Fetch the [=verifiable credential=] from the id of the CredentialRefresh object.

JsonSchemaValidator2019

The "JsonSchemaValidator2019" Data Schema Validation Method points to a [[JSON-SCHEMA-2019-09]] file that can be used by a [=verifier=] to determine if the [=verifiable credential=] is well formed. This extension is used to complete the "data validation" step described in [[[#verification]]].

Validating JSON Schema

If the credentialSchema property is present, and the type of the CredentialSchema object is "JsonSchemaValidator2019", the [=verifier=] SHOULD follow these steps:

  1. Fetch the [[JSON-SCHEMA-2019-09]] file from the id of the CredentialSchema object.
  2. Serialize the credential into a JSON string.
  3. Use a JSON Schema Validator compatible with [[JSON-SCHEMA-VALIDATION-2019-09]] (home grown or off-the-shelf) to validate the credential's serialized JSON string.
