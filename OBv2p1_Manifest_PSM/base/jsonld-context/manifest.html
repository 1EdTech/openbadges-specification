Class Definitions

"dtBadgeConnectAPI" Class

The list and definition of the child properties is:-

name
The name of the platform supporting the API. This SHOULD reflect the user-facing identity of the platform requesting authorization.
image
An image representing the platform. May be a URI to a hosted image or a Data URI.
apiBase
Fully qualified URL that will be concatenated with the API endpoints. It SHOULD NOT have a trailing slash '/'. E.g. apiBase + '/assertions'.
version
A string representing the implemented version. MUST be in the format of vMAJORpMINOR where MAJOR and MINOR are integers.
termsOfServiceUrl
Fully qualified URL to the platform's terms of service. Other platforms SHOULD link to this resource as part of their authorization interface.
privacyPolicyUrl
Fully qualified URL to the platform's privacy policy. Other platforms SHOULD link to this resource as part of their authorization interface.
scopesOffered
Applies to Hosts only. An array of strings listing the scopes supported by the Host in the form of fully qualified URLs to the scope descriptors.
scopesRequested
Applies to Relying Parties only. An array of strings listing the scopes supported by the relying party in the form of fully qualified URLs to the scope descriptors.
authorizationUrl
Applies to Hosts only. A fully qualified URL to the host's authorization endpoint.
tokenUrl
Applies to Hosts only. A fully qualified URL to the host's token request endpoint for exchanging an authorization code for a bearer token.
redirectUris
Applies to Relying Parties only. An array of one or more fully qualified URIs representing the valid redirect values for authorization requests. The HOST MUST reject any authorization requests with redirect URIs that are not an exact match of a registered URI.
keys
Applies to Relying Parties only. Allows the relying party to declare the location of their JWKS file. Currently this file MUST be named jwks.json and placed in the .well-known directory. E.g. https://YOUR_DOMAIN/.well-known/jwks.json.
